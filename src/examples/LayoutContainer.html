Example "Playground"
A button usually triggers an action by pressing or clicking it. Use short and distinct labels to make clear what action the button triggers. The button's label can be extended by an icon, supporting the label visually. Icons can also be used instead of labels or replace a part of the label, e. g. by using '+ new file' instead of 'add new file'. Make sure the icons that are used are universally understood/recognised.

Parameter Type Description
disabled boolean Disabled state of the control
label string The label of the button

    data-context="workflow"
    (clicked)="buttonClicked()">
